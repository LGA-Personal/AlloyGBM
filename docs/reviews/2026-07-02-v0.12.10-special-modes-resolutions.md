## 1. Critical correctness issues

### 1.1 Row subsampling / GOSS committed predictions

**Fixed in pending v0.12.11 PR by `9d93ad6`.** The single-output and multiclass training loops now rebuild
candidate training predictions by walking the accepted tree over every training row after tree
construction. This keeps the committed training state consistent with artifact inference for
uniform row subsampling, GOSS, DART, missing-value routing, native categorical routing, and PL
leaf evaluation.

### 1.2 DART training-loss gate

**Fixed in pending v0.12.11 PR by `9d93ad6`.** The training-loss acceptance gate now exempts DART in the
same spirit as ranking objectives. DART dropout and normalization are non-monotone by design,
so a higher training loss in a round is not a valid reason to discard the round or terminate
the fit.

### 1.3 PL leaf conditioning

**Partially fixed in pending v0.12.11 PR.** PL leaf solves now standardize selected
regressor values internally before accumulating `Xᵀg` and `XᵀHX`, persist the
per-leaf scale metadata in a v2 `LinearLeafCoefficients` payload, and decode old
v1 linear-leaf artifacts with identity scaling. Prediction, SHAP attribution,
and native artifact loading all evaluate the same standardized leaf terms, with
NaN regressors contributing the mean-imputed standardized value (`z = 0`).

The same PR also switches PL regressor selection from `features[0..8]` to the
leaf split path, capped at `MAX_PL_REGRESSORS = 8`.

## 5. Factor neutralization

**Partially fixed in pending v0.12.11 PR.** Objective-dependence is now documented: per-round
gradient neutralization projects gradients while leaving Hessians unchanged, so non-constant
Hessian objectives do not produce a simple projection of unneutralized leaf values. Fitted scalar
estimators and joint `MultiLabelGBMRanker` now add post-fit prediction exposure diagnostics to
`factor_exposure_diagnostics_`: `prediction_exposure_dot`, `prediction_exposure_abs`, and
`prediction_exposure_l2`, computed from transformed fit-time exposures and fitted training
predictions. Multiclass models have no single scalar prediction vector, so they keep the
transform/means/stds metadata but omit the prediction-exposure entries.

**Further narrowed in pending v0.12.11 PR.** Active `split_penalty`
neutralization now resolves the default `factor_exposure_transform="none"` to an effective
`"standardize"` transform when `factor_penalty > 0`, while `per_round_gradient`, `pre_target`,
and inert split-penalty configurations still default to no transform.

**Further narrowed in pending v0.12.11 PR.** Scalar CPU native-categorical `split_penalty`
candidates now share the numeric path's prefix-sum treatment. The backend precomputes per-category
factor exposure sums once in the Fisher-sorted category order, then advances the prefix in O(K)
per candidate instead of rescanning node rows for each prefix and missing-direction candidate.

**Further narrowed in pending v0.12.11 PR.** Scalar CPU factor split scoring now reuses a
thread-local `FactorSplitScratch` buffer for each feature scan when a factor context is active,
matching the existing histogram arena pattern and avoiding per-feature scratch Vec allocation
churn.

**Further narrowed in pending v0.12.11 PR.** `FactorProjector` now parallelizes the fit-time
Gram build, per-round RHS accumulation, and residual-write passes over rows. Repeated scalar,
multiclass, and joint per-round gradient projection call sites now pass a reusable residual
scratch buffer instead of allocating one projection vector per output/round.

## 6. Objectives

### 6.1 Ranking gradient optimizations

**Partially fixed in pending v0.12.11 PR.** Ranking objective gradient computation now runs
query groups independently with Rayon for `rank:pairwise`, `rank:ndcg`, `rank:xendcg`, and
`yetirank`. NDCG-weighted pairwise objectives also precompute per-document gains and
rank discounts once per group/per permutation instead of recomputing `powf` and `log2`
inside each pair loop. `LambdaMARTObjective::compute_gradients_into` now fills the caller's
buffer directly rather than allocating a temporary gradient vector, while preserving the
existing `GradientPair::new` validation behavior.

**Further narrowed in pending v0.12.11 PR.** Single-output `GBMRanker` now exposes
`ranking_sigma` for pairwise ranking-family objectives. The Rust `rank:pairwise`,
`rank:ndcg`, and `yetirank` objectives validate positive finite sigma values and apply the
standard logistic sharpness scaling to pairwise gradients and Hessians, while defaulting to
`1.0` for backward-compatible behavior.

**Further narrowed in pending v0.12.11 PR.** Joint `MultiLabelGBMRanker` now accepts the same
`ranking_sigma` kwarg when `multi_label_mode="joint"`. The PyO3 bridge forwards the positive
finite sigma into joint `rank:pairwise` and `rank:ndcg` objectives, and the Rust joint objective
variants carry it through per-output gradient computation. The default remains `1.0`, preserving
existing joint ranking fits.

**Further narrowed in pending v0.12.11 PR.** `GBMRanker` and joint
`MultiLabelGBMRanker` now expose `lambdarank_truncation_level` for `rank:ndcg`. When set, the
LambdaMART pair loop only scores pairs where at least one document is currently in the top-k
positions for its query. `None` preserves the previous all-pairs behavior, and invalid values
(`0` or non-integers) are rejected at the Python boundary and in the Rust objective constructor.

**Further narrowed in pending v0.12.11 PR.** `GBMRanker` and joint
`MultiLabelGBMRanker` now expose `lambdarank_normalize` for `rank:ndcg`. When enabled, each query's
LambdaMART pair weights use score-distance regularization and the accumulated lambdas/Hessians are
rescaled by the query-local `log2(1 + sum_lambdas) / sum_lambdas` factor. The default remains
`False` for backward-compatible training behavior.

### 6.2 GLM stabilizers and f64 accumulators

**Partially fixed in pending v0.12.11 PR by `97ed521` and follow-up PR by
[`abc5ef7`](https://github.com/LGA-Personal/AlloyGBM/commit/abc5ef7) (merged with clippy-cleanup
[`4b58f7c`](https://github.com/LGA-Personal/AlloyGBM/commit/4b58f7c)).** GLM initial predictions
now compute weighted target means with f64 accumulators, and Poisson/Gamma/Tweedie objective
losses now accumulate telemetry in f64 before returning f32. The Poisson objective also inflates
its Newton hessian by `exp(0.7)` by default, matching the standard Poisson max-delta-step
stabilizer, and exposes `poisson_max_delta_step` on `GBMRegressor`, `GBMRanker`, independent
and joint `MultiLabelGBMRanker`, and the native Rust `TrainParams`/objective path.

The tunable step is carried as a `max_delta_step` field on `PoissonObjective` itself (defaulting
to 0.7 via `Default`); the initial parallel `TunedPoissonObjective` type was removed in `4b58f7c`
because its `PoissonObjective::new(..)` returned a different type and tripped
`clippy::new_ret_no_self` under `-D warnings`. `new(..)` now returns `Self`.

## 7. Joint multi-output trainer

**Partially fixed in [PR #75](https://github.com/LGA-Personal/AlloyGBM/pull/75) by
[`227cfb8`](https://github.com/LGA-Personal/AlloyGBM/commit/227cfb8).** Joint
`MultiLabelGBMRanker` now validates native-categorical columns before training when
`max_cat_threshold > 0`: category IDs must be integer-valued, dense `0..K-1`, and no joint
native-categorical feature may exceed 64 categories. This makes the joint trainer's `u64`
categorical bitset cap explicit instead of silently letting Rust skip high-cardinality native
categorical splits.

**Further narrowed in pending v0.12.11 PR.** Joint native-categorical `split_penalty` scoring now
mirrors the scalar categorical prefix-sum fix: level-wise and leaf-wise joint categorical split
searches precompute per-category factor exposure sums once in Fisher-sort order, then apply the
K-output factor-load penalty during each prefix candidate score. This lets the penalty choose a
different categorical partition instead of subtracting a row-scanned penalty only after the
unpenalized best partition was chosen.

**Further narrowed in pending v0.12.11 PR.** The review-requested joint `row_subsample=0.5`
A/B is now pinned by a Rust regression: sampled joint training runs all requested rounds, then
artifact predictions on every training row must beat a mean predictor and stay within the
target range envelope. The test verifies the existing joint round-end full-tree walk over all
rows, so no trainer behavior change was required for that path. The per-round K-output gradient
phase now also parallelizes across outputs with Rayon while preserving output order and
objective error propagation.
