### 2.2 Release the GIL

**Partially fixed in [PR #78](https://github.com/LGA-Personal/AlloyGBM/pull/78) and follow-up
PRs.** Native prediction paths now release the Python GIL around Rust compute via
`Python::detach`, including `NativePredictorHandle`, the standalone predictor bridge functions,
and the joint multi-output predictor handle. Native SHAP bridge calls now also detach around
explanation and global-importance compute, including scalar, multi-output, dense, and
predictor-aligned binning entry points.

Native training bridge calls now detach around the Rust fit, including fits that use Python
custom objectives, custom losses, or custom metrics. Python argument parsing and bytes/list
materialization still happen before detaching; `CustomPythonObjective` and
`CustomPythonMetricCallback` reattach with `Python::attach` only while invoking the Python
callable and extracting its result.

### 2.9 Move quantile-cut derivation out of Python

**Partially fixed in [PR #73](https://github.com/LGA-Personal/AlloyGBM/pull/73) by

## 5. Robustness / validation

**Partially fixed in pending v0.12.11 PR.** `ModelMetadata` JSON decoding is now key-driven for
the current metadata schema, so artifacts no longer depend on the serializer's exact field order.
Required fields (`format_version`, `feature_names`, `trained_device`) are validated by key, while
`objective` and `num_classes` keep their existing backward-compatible defaults. Replacing the
remaining hand-rolled parser with `serde_json` and accepting unknown future fields remains open.

**Partially fixed in pending v0.12.11 PRs.** Predictor artifact loading now bounds the per-tree
heap-style node slot array before allocation. Artifacts whose sparse local node IDs would require
more than `MAX_TREE_NODE_SLOTS` (65,536) predictor node slots for a single tree are rejected with a
contract violation instead of constructing a large mostly-empty `nodes_by_local_id` array.

The limit is a single shared constant, `alloygbm_core::MAX_TREE_NODE_SLOTS`, enforced on **both**
sides of the train/predict boundary so the contract is symmetric: the trainer's
`encode_tree_node_id` rejects any split whose heap child would land at slot
`>= MAX_TREE_NODE_SLOTS`, so a fit that would exceed the budget fails fast with a clear error
rather than emitting an artifact the predictor cannot load. This closes the asymmetry flagged in
review — previously the trainer permitted local node IDs up to the encoding stride (`1 << 20`),
16× larger than the predictor's cap, so a deep-but-legal tree could train yet fail to load. The
stride (`1 << 20`) is retained purely as the per-tree id multiplier; the slot budget is the
tighter, deliberately conservative bound (≈ depth 16, far more leaves than any realistic tree).

Artifact loading now also rejects excessive declared section counts and per-section payload
lengths before descriptor parsing or payload copying. The serializer and `ModelIoContractV1`
validator enforce the same limits so locally emitted artifacts cannot exceed the loader budget.

Additional artifact hardening remains open, including bin-count limits and broader metadata
validation.
