# AlloyGBM Docs

This repository now uses a simple documentation layout:

- `roadmap/`
  - Current project direction and medium-term priorities.
- `benchmarks/`
  - Benchmark framing and evaluation notes.
- `ideas/`
  - Research notes, inspiration, and candidate follow-ups.
- `reference/`
  - Stable pointers into the codebase and developer-facing entry points.
- `archive/`
  - Legacy material from the previous version-layer planning system.
- `architecture/`
  - Transitional location for a small amount of not-yet-moved legacy material.

If you are trying to understand the project today, start here:

1. `docs/roadmap/current.md`
2. `benchmarks/README.md`
3. `docs/reference/README.md`

The old nested architecture tree is intentionally kept for historical context, but it is no longer the active planning model for the repository.
