  - malformed required-section layouts now fail with aligned compatibility-mode diagnostics.
- Added/updated tests:
  - core tests for payload-start and contiguous-offset contract enforcement,
  - engine tests for deterministic compatibility-mode error text on malformed required-section layouts,
  - predictor tests for deterministic compatibility-mode error text on malformed required-section layouts.

## Non-Intuitive Decisions
- Decision: enforce contiguous section offsets, not just non-overlap.
- Reason: serializer already emits contiguous payload sections; enforcing this in validation prevents ambiguous descriptor layouts and reduces parse-surface variability.
- Impact: malformed artifacts that previously relied on descriptor gaps are now rejected deterministically.

- Decision: centralize compatibility-error string formatting in `core`.
- Reason: both `engine` and `predictor` need the same required-section compatibility semantics; centralization prevents diagnostic drift.
- Impact: compatibility failures now present aligned section-count context regardless of ingest path.

## Migration Notes
- Legacy trees-only artifacts remain supported only in compatibility mode paths.
- Artifacts with malformed required-section layouts (duplicate/missing required sections or non-contiguous descriptor offsets) now fail with deterministic compatibility diagnostics.
- For external artifacts that fail new offset-contract validation, migration path is:
  1. import via a trusted pipeline that can decode and validate trees,
  2. re-emit using `TrainedModel::to_artifact_bytes()` to produce canonical contiguous-layout v1 artifacts.
